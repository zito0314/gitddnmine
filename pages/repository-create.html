Repository Request

Repository 생성 요청

신규 Repository는 조직 정책에 따라 관리자 승인 후 생성됩니다. 요청 정보, Repository명, 템플릿, 멤버를 입력해 요청을 제출하세요.

기본 정보

Repository 생성 요청에 필요한 제목, 내용, Repository명을 입력합니다.

반드시 소문자 또는 대문자, 숫자, 이모티콘, 밑줄로 시작해야 합니다. 마침표, 더하기 기호, 대시, 공백을 포함할 수도 있습니다.

템플릿

Repository 생성 시 적용할 템플릿을 선택합니다.

멤버

Repository 생성 후 바로 접근해야 하는 멤버를 지정합니다.

요청 처리 흐름
1 요청자가 Repository 정보를 입력하고 생성 요청을 제출합니다.
2 관리자가 요청 내용, Repository명, 템플릿을 검토합니다.
3 승인 완료 후 Repository가 생성되고 멤버에게 접근 권한이 부여됩니다.
권장 입력 기준

Repository name은 서비스나 모듈의 역할이 드러나도록 작성하는 것을 권장합니다. 예: mobile-banking-api, common-ui-components

적용될 수 있는 정책
MR Approval Pipeline Security Check Audit Trail
✓

Repository 생성 요청이 제출되었습니다.

요청은 관리자 검토 후 승인 상태로 변경됩니다. 승인 완료 시 Repository 목록에서 상세 화면으로 진입할 수 있습니다.
