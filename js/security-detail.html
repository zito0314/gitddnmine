SQL Injection
취약점 개요

사용자 입력이 SQL 쿼리에 직접 삽입되면 공격자가 악의적인 SQL 구문을 주입해 데이터베이스를 조작할 수 있습니다.

수정 방향
1 Statement 대신 PreparedStatement를 사용하세요.
2 모든 사용자 입력값에 파라미터 바인딩을 적용하세요.
3 ORM 프레임워크(JPA/MyBatis)의 파라미터 바인딩 기능을 적극 활용하세요.
