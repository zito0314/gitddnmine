# 2026-05-19 작업 요약

오늘은 `gitddn`의 Pipeline 관련 화면을 정리하고, Repository/MR 화면과 같은 제품 스타일로 맞췄다.

## 진행한 내용

1. Pipeline 목록 화면을 새로 정리했다.
   - 10개 항목 노출
   - `Running / Failed / Finished` 탭 필터 동작
   - `재실행` / `실행 취소` 상태 전환
   - Branch와 Commit은 각각 파일 화면, Commit 상세로 연결

2. Pipeline 상세 화면을 제품 스타일에 맞게 바꿨다.
   - hero, summary card, tab 구조를 MR 상세와 같은 톤으로 통일
   - `Stage`와 `Job dependencies`를 같은 영역에서 전환 가능하도록 구성
   - dependency hover 시 연관 Job 하이라이트 추가

3. Job 상세와 Commit 상세를 추가/정리했다.
   - Job 로그, 실행 정보, 관련 Job 패널 구성
   - Commit 상세 페이지 추가
   - 상세 화면 전반의 카드, 칩, 테이블 스타일을 통일

4. GNB와 이동 경로를 정리했다.
   - Pipeline 관련 화면 전체에 Repository 상세와 같은 GNB 적용
   - Pipeline 화면의 breadcrumb 제거
   - 저장소 상세의 Pipelines 진입 시 새 Pipeline 목록으로 연결되도록 수정

## 결과

- Pipeline 흐름이 목록 → 상세 → Job 상세로 자연스럽게 이어지게 됐다.
- 기존 Repository / MR 화면과 시각적 톤을 맞춰서, 화면마다 느낌이 크게 달라지지 않도록 정리했다.
- 변경 내용은 여러 커밋으로 나눠 기록해 두었다.
